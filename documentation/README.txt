Ce dossier contient tout un tas de documentation sur l'application SPEAT.

-> Le dossier "specification" contient des éléments de spécification 
concernant l'application.
Spécification = Ce que doit faire l'application, ses fonctionnalités, 
les besoins auxquels elle répond...
Ce dossier pourra donc contenir des documents comme un cahier des charges, 
un index de fonctionnalités, etc...

-> Le dossier "conception" contient des éléments de conception de 
l'application.
Conception = comment l'application implémente les fonctionnaltés demandées ?
Comment l'application répond-elle aux besoins ?
Comment l'application est-elle structurée, comment fonctionne-t-elle, etc...
On pourra donc trouver dans ce dossier, des documents comme des diagrammes de 
classes, des diagrammes de séquence, diagrammes d'objets, éventuellement 
la définition de tests unitaires et d'intégration, etc...

-> Le dossier "guide" contient des guides spécifiques sur l'application client 
(front end) et l'API (back end) destinés aux développeurs.
REMARQUE : Le guide de développement de l'application client 
(client-app-guide.md) est fait pour être consulté sur GitHub, dans la 
continuité de la documentation exposée sur GitHub.

-> Le dossier "autres-ressources" contient des ressources utiles en vrac pour 
les développeurs front et back.

-> Le dossier "images" contient des images utilisées dans les fichiers de 
documentation (les fichiers Markdown).
